WTF?

A 'CoinTokens' contract (template coded by Matto, implemented here for $LFG) allows back and forth exchange between an ERC-20 and ERC-1155 NFTs that represent integer denominations of that ERC-20. The contract template can be used for any ERC-20 token, and for user security, ERC-20 contract and ERC-1155 denomination parameters are locked at deployment. The contract has on-chain royalty functions for the ERC-1155s and does not implement any operator filters. Also, there is a customization option built in: it can return ERC-20 'change' leftover from denomination functions or it can lock it permanently to the contract. Any number of integer denominations are supported, always with a tokenID matching the denomination.



LFGCoinToken Specifc Info


This implementation includes 8 denominations that are exchangable for $LFG, as seen in the following list.

These denominations are represented as ERC-1155 NFTs where the tokenID matches the denomination (eg. tokens with ID '69420' can be exhanged for exactly 69,420 $LFG).

Only these 8 denominations can be exchanged for $LFG, but note that there is another token with 0 value: tokenID 0. That token is a 1:1 burner token, received for burning/locking $LFG to the contract. It cannot be exchanged for $LFG.

There are no fees for exact exchanges between ERC-20 and ERC-1155 tokens (other than gas), but any ERC-20 overage amount used in Denominate functions is permanently locked to the contract.

Denominate Functions ($LFG → LFGCoinToken NFTs)

Denominate All automatically denominates the submitted amount of $LFG into the largest possible denominations and allocates ERC-1155 tokens accordingly. Any leftover $LFG (any amount less than 69 - the smallest denomination), is locked to the contract and cannot be retrieved by anyone.



Denominate Into Token denominates the submitted amount of $LFG into the submitted denomination (if valid) and allocates ERC-1155 tokens accordingly. Any leftover $LFG (any amount less than the submitted denomination), is locked to the contract and cannot be retrieved by anyone (if you denominate into token 1000000, and send in 1,999,999 LFG, you lock 999,999 LFG to the contract, forever).


Did you try it? Check your NFT collection!



Liquidate Functions (LFGCoinToken NFTs → $LFG)

Liquidate All automatically liquifies (burns) ALL denominated LFGCoinTokens owned by the user and returns their value in $LFG.



Liquidate From Token Liquifies a submitted amount of a submitted denomination of LFGCoinToken and returns their value in $LFG.

LFG CoinToken FAQ (click to expand)
Can anyone withdraw $LFG held or 'locked' by the contract?

The contract should only send $LFG to an account if LFGCoinTokens are liquidated from that account.


Has this contract been audited?

No. While I believe I coded it well to protect from hacks, please consider it a risky experiment with no warranties.


Cost? Fees? Royalties?

There is no cost besides gas, and there are no 'fees' charged on ERC-20/ERC-1155 exchanges... But for fun (and to reduce the available supply of $LFG by locking/burning coins) any $LFG overages submitted in denominate functions are locked to the contract. Oh, and the ERC-1155 tokens have 3% royalties payable to artist.MonkMatto.eth.

The contract template used here is MIT licensed, so you can go nuts with it. If you want to toss me a tiny bag of whatever ERC-20 you use the contract with, I won't be upset about it :D MonkMatto.eth pls!
